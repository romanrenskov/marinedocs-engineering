# plans/

Папка `plans/` хранит технические планы реализации функций.
Это рабочее место для подготовки, проверки и сопровождения изменений.
Один план описывает ровно одну функцию.
Если подходящий план уже существует, работа продолжается в нем.
Новый план создается только для новой самостоятельной функции.
Имя файла плана должно иметь формат `YYYY-MM-DD-название-функции.md`.
Дата в имени берется текущая: год, месяц и день создания плана.
Название функции пишется коротко, через дефисы, без пробелов.
Каждый план делится на фазы реализации.
У каждой фазы должен быть статус: `[ ]` не выполнено или `[x]` выполнено.
Фазы должны быть достаточно конкретными, чтобы их можно было проверять.
В конце плана должен быть итоговый блок.
В итоговом блоке указывается, реализована функция целиком или нет.
Там же кратко фиксируется, что осталось сделать.
Любой агент, работающий с планом, обязан актуализировать его после каждой сессии.
Рабочий цикл: первый чат формирует план.
Второй чат разбивает план на фазы.
Третий чат проверяет каждую фазу на адекватность.
Только после этого начинается реализация.
